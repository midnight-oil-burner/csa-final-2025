Unit 3: Boolean Expressions and if Statements

3.1: Boolean Expressions

Certain expressions which help you with comparing values are used in writing Boolean statements: 

    boolean one = a == b; // true if a is equal to b; don't use this on doubles
    boolean two = a != b; // true if a is NOT equal to b
    boolean three = a < b; // true if a is less than b 
    boolean four = a > b; // true if a is greater than b 
    boolean five = a <= b; // true if a is less than or equal to b
    boolean six = a >= b; // true if a is greater than or equal to b

An expression containing any of the above evaluates to a boolean.

3.2: if Statements and Control Flow

if statements check whether or not a boolean expression is true, and executes a block of code if it is: 

    if(booleanExpression)
    {
        // code here is run if booleanExpression is true
    }

if is a keyword, so don't name your variables "if." Note that the first line is akin to a method signature; there's no semicolon. Also much like a method, the evaluation of a boolean expression interrupts the execution of code. The code inside the scope of an if statement is passed over if the boolean is false. Also, if an if statement's contents is EXACTLY one line, you don't have to use curly brackets, though it's really easy for this to cause an error if you forget.

A quirk with return statements inside the scope of an if is that you'll probably need multiple return statements, consider the following method: 

    public static boolean isCool(String name) {
        if(name.equals("Ms. Zhu"))
            return true;
    }
    // the above results in a compilation error, as there is no return statement if the name isn't "Ms. Zhu."

    public static boolean isCool(String name) {
        if(name.equals("Ms. Zhu"))
            return true;
        return false;
    }
    // the above works perfectly fine (it also works if the additional line follows an else statement)

    public static boolean isCool(String name) {
        if(name.equals("Ms. Zhu"))
            return true;
        if(!name.equals("Ms. Zhu"))
            return false;
    }
    // the above will not work.

    // note that the above method can be simply reduced to "return name.equals("Ms. Zhu");"

3.3: if-else Statements

else statements can be added after if statements. They represent code that will be run if the if statement's boolean happens to be false: 

    public static boolean isCool(String name) {
        if(name.equals("Ms. Zhu"))
            return true;
        else
            return false;
    } 
    // works exactly the same as the previous method

One line bracket omission works the same as if statements. They are also not required in order for an if statement to compile, and each if statement can only have one associated else statement.

3.4: else if Statements

else if statements can be added after if statements to represent multiple boolean expressions. 

    public static boolean isCool(String name) {
        if(name.equals("Ms. Zhu"))
            return true;
        else if(name.equals("Joe Cool"))
            return true;
        else 
            return false;
    }

As you can see, else-if expressions also require boolean expressions, and the one line bracket omission also applies. These are also optional in making an if statement compile, and there is no limit to the amount of else-ifs you can have per if statement.

3.5: Compound Boolean Expressions

Certain symbols can chain or boolean expressions together or alter them: 

    Boolean hoo = a && b; // true if a and b are both true, false otherwise
    Boolean joo = a || b; // true if either a or b are true, false if both are false
    Boolean greg = !a; // true if a is false, false if a is true

&& is evaulated before ||: 

    Boolean foop = a && b || c && d;
    // effectively means:
    Boolean foop = (a && b) || (c && d);

As you'd expect, boolean expressions inside parentheses are executed first.

&& and || use short circuit evaluation; Java doesn't check the second expression if it can tell by looking at the first expression whether the compound boolean will be true or false. E.g., in the above boolean hoo, if a is false, it doesn't check b as hoo will be false regardless.

3.6: Equivalent Boolean Expressions

DeMorgan's Laws can be applied to boolean expressions to find an equivalent boolean: 

    !(a && b) is equivalent to !a || !b
    !(a || b) is equivalent to !a && !b
    !(!a) is equivalent to a

    !(c == d) is equivalent to c != d
    !(c != d) is equivalent to c == d
    !(c < d) is equivalent to c >= d
    !(c > d) is equivalent to c <= d
    !(c <= d) is equivalent to c > d
    !(c >= d) is equivalent to c < d

All that's being done is essentially the distribution of the !, like how you would distribute a number in math.

3.7: Comparing Objects

Again, you should almost always use an equals method instead of == to check if two objects are meaningfully equivalent. However, == lets you check if two object references are aliases--different names referring to the same object in memory: 

    String str1 = new String("yo");
    String str2 = str1; // str2 is now an alias for str1
    System.out.println(str1); // prints "yo"
    System.out.println(str2); // prints "yo"
    System.out.println(str1 == str2); // prints "true," so they're aliases
    System.out.println(str1.equals(str2)); // prints "true"
    str1 = "test";
    System.out.println(str1); // prints "test"
    System.out.println(str2); // prints "yo"; Strings are immutable!

A common technique when debugging is using == to compare objects with null to see if they actually exist.
